# UI and Mobile Rule

- A working component is not complete until hierarchy, spacing, readable copy, loading/error/empty states and mobile behavior are considered.
- Preserve the product's chosen visual language; do not drop in generic AI-dashboard styling.
- Check narrow viewport behavior for any changed interaction.
- Prefer visible, direct interaction feedback over hidden cleverness.
- For visual changes, verify through the running app or a rendered view when available.
