# Optional Project Profiles

Copy only the relevant additions into the project `CLAUDE.md` or rules.

## Web product / application

- Require real viewport testing of every altered user journey.
- Accessibility, responsive layout and error states are completion requirements.
- Prefer a small polished workflow over a dashboard full of inert controls.

## Game / interactive simulation

- Separate deterministic simulation state from rendering/presentation transforms.
- Require a playable loop check after gameplay changes.
- Inspect mobile controls, readability and performance budget before polish claims.

## Research / synthetic cognition system

- Lock experiment claim and success criteria before running tests.
- Require falsification, baselines and evidence-language alignment for result claims.
- Keep instrument/simulator behavior distinct from interpretation and narrative.

## Agent/tool system

- Every tool gets an explicit trust boundary and failure behavior.
- Validate inputs and constrain permissions before expanding autonomy.
- Treat traceability, recovery and human approval points as product features.
